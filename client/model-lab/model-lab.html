Quick Start

Get your first model running in 60 seconds:

Step 1 Go to the Train tab and paste text into the text box — a paragraph from a book, some song lyrics, a chunk of code, anything with 500+ characters.
Step 2 Leave the default settings (they're chosen to work well for a first run). Or read the Hyperparameters section below to customize.
Step 3 Click Train and watch the console. You'll see the model go from random gibberish to recognizable text patterns. The whole run takes 1–3 minutes.
Step 4 When training finishes, click Generate to produce new text. Change the Temperature slider and generate again — notice how the output changes.
Step 5 Read the milestones and experiment suggestions the console printed. Try one! Change a setting and train again.
Note The model trains entirely in your browser using your GPU (via TensorFlow.js). Nothing is sent to any server. Your text stays private.

What is a Language Model?

A language model is a system that learns to predict the next piece of text given what came before. When you type on your phone and it suggests the next word — that's a language model. ChatGPT is the same idea, massively scaled up.

In this lab you're building the simplest version: a character-level model. Instead of predicting whole words, it predicts the next character. Given "hel", it should learn to predict "l" (to form "hell" or "hello").

Key Insight Every modern AI text generator — GPT-4, Claude, Gemini, LLaMA — uses the same core loop: predict the next token, check the error, update weights. The differences are in scale and architecture, not the fundamental idea.

The Training Pipeline

Here's exactly what happens when you click Train:

1. Tokenization

Text is converted to numbers. We scan your text, find every unique character, sort them, and assign each an integer ID.

Example: if your text contains the characters a b c d e (space), the mapping might be: space=0, a=1, b=2, c=3, d=4, e=5.

Note GPT models use subword tokenization (BPE) with ~100,000 tokens like "un", "believ", "able". We use single characters for transparency — you can see exactly what the model sees.

2. Embedding

Each character ID is converted into a vector — a list of 32 numbers. Initially random, but during training the model learns to position similar characters nearby in this 32-dimensional space.

Why? Because a single integer (like 5) gives the model no information about relationships. A vector lets the model learn that a and e are both vowels and should behave similarly.

3. LSTM (Long Short-Term Memory)

The core of the model. It reads the sequence of embedded characters one by one, maintaining a hidden state — a vector of numbers that acts as its "memory" of everything it's read so far.

At each step, the LSTM decides:

  • Forget gate — what to erase from memory (e.g., a pattern that's no longer relevant)
  • Input gate — what new information to store (e.g., "we just saw a vowel")
  • Output gate — what to output for this step (what's useful for predicting the next character)
Key Insight The "Long Short-Term Memory" name comes from solving the vanishing gradient problem. Simple RNNs forget things after ~10 steps. LSTMs can remember patterns across hundreds of steps thanks to their gating mechanism.

4. Dense Layer + Softmax

The LSTM's output is a hidden vector. The Dense layer converts it into vocabSize numbers — one score per possible next character. Softmax then converts these scores into probabilities that sum to 1.

Example output: a: 0.35, b: 0.02, c: 0.01, …, space: 0.28 — the model thinks "a" or "space" is most likely next.

5. Loss & Backpropagation

We compare the model's predicted probabilities to the actual next character using cross-entropy loss. If the model predicted 0.02 for the correct character, the loss is high. If it predicted 0.95, the loss is low.

Then backpropagation calculates how each weight contributed to the error, and the optimizer (Adam) nudges every weight slightly in the direction that reduces the loss. One "step" done.

Key Insight Training is just optimization. We have a giant math function (the model) with thousands of adjustable knobs (weights). We repeatedly ask "how wrong are you?" (loss) and "which knobs should I turn, and how much?" (gradients via backprop). Repeat thousands of times.

Reading the Console Output

The training console uses color-coded tags. Here's what each one means:

[info] System status [learn] Explanation [train] Training step [milestone] Achievement [weights] Parameter stats [memory] GPU usage [sample] Generated text [warn] Problem detected [experiment] Suggestion [error] Error

Key metrics to watch

  • Loss — the model's error. Lower = better. Your starting loss ≈ ln(vocabSize) (random guessing). Good training gets it well below that.
  • Perplexity — eloss. Means "the model is as uncertain as if choosing between N options." Starting perplexity = vocabulary size. A perplexity of 5 is very good for a tiny model.
  • Weight stats — μ (mean) near 0 is healthy. σ (standard deviation) shows specialization. If σ is tiny after many steps, that layer might not be learning. If σ is very large (>1), the model may be unstable.
  • Samples — the most intuitive metric! Early samples are gibberish. As training progresses, you'll see common letter pairs, then words, then phrases emerge. Compare samples from step 1 vs step 200.
Note — What is "normal"? For typical English text with defaults (128 hidden, lr 0.003): loss starts ~3.5–4.0, should reach ~1.5–2.5 after 300 steps. If loss stays above 3.0 after 100 steps, you may need more training text or different settings.

Hyperparameters Deep Dive

Hidden Units (LSTM size)

Controls the model's capacity — how complex the patterns it can learn.

  • 32–64 — Very small. Learns basic character frequencies and common bigrams (two-letter pairs). Fast to train.
  • 128 — Default. Can learn word-level patterns and short phrases. Good balance of speed and quality.
  • 256–512 — Large. Can capture longer dependencies and rarer patterns. Needs more training data and steps to work well, or it may overfit.
Key Insight More parameters is not always better. A 512-unit model trained on 200 characters will memorize the text (overfit) rather than learn patterns. Match model size to data size. Rule of thumb: your text should have at least 10× more characters than the model has parameters.

Learning Rate

The most important hyperparameter. It controls how much weights change per step.

  • 0.01–0.1 — Aggressive. Model learns fast but may overshoot and jump around or diverge.
  • 0.001–0.005 — Sweet spot for most experiments here. Learning is visible within the first 20 steps.
  • 0.0001 — Very conservative. Loss drops slowly but smoothly. Useful if you have many steps (1000+).
Warning If you see [warn] Loss jumped! in the console, your learning rate is likely too high. Halve it and try again.

Sequence Length (Context Window)

How many previous characters the model sees when predicting the next one. This is equivalent to the "context window" in LLMs.

  • 10–20 — Model can only see nearby characters. Good enough for learning common words but can't capture sentence-level patterns.
  • 40 — Default. Sees about half a sentence. Good for learning word boundaries and phrase patterns.
  • 80–128 — Sees full sentences. Better for structured text like code (needs matching brackets, indentation). Uses more memory.

Temperature

Only affects text generation, not training. Controls the randomness of character selection.

  • 0.1–0.3 — "Greedy." Almost always picks the highest-probability character. Output is repetitive but coherent.
  • 0.7–0.9 — Balanced. Picks likely characters with some variation. Usually the best-sounding output.
  • 1.0 — Unmodified probabilities. The "true" model distribution.
  • 1.5–2.0 — "Creative." Flattens the distribution so unlikely characters get chosen more. Output gets wild and often nonsensical.

Mathematically: probabilities are divided by temperature before softmax. temp < 1 makes the distribution peakier; temp > 1 makes it flatter.

Hands-On Exercises

Each exercise teaches a core ML concept. Try them in order.

Exercise 1: Your First Model
Paste a paragraph of English text (300+ chars). Keep all defaults. Click Train.
Observe: How quickly does loss drop? At what step do recognizable words first appear in the [sample] output? Write down the step number.
Exercise 2: Learning Rate Crash Course
Train the same text three times with learning rates: 0.1, 0.003, 0.0001.
Observe: At 0.1 the loss likely jumps or explodes. At 0.003 it drops steadily. At 0.0001 it barely moves in 300 steps.
Lesson: Learning rate is a tradeoff between speed and stability. Too high = chaos. Too low = wasting compute.
Exercise 3: Overfitting vs Generalization
First: paste only 100 characters. Train 500 steps with 256 hidden units. Second: paste 3000+ characters with the same settings.
Observe: With tiny data, loss hits near-zero and samples are exact copies of the input — the model memorized it (overfit). With more data, loss is higher but samples are original text.
Lesson: More data forces the model to learn patterns, not specific text. This is the #1 insight in ML.
Exercise 4: Temperature Exploration
After training, click Generate with temperature 0.2. Then change to 0.8. Then 1.5. Then 2.0.
Observe: Low temp → repetitive and safe. High temp → creative but increasingly garbled.
Lesson: This is the same "temperature" slider used in ChatGPT. There's no "right" value — it depends on the task.
Exercise 5: Architecture Comparison
Train the same text with hidden units: 32, 128, and 512. Same steps (300) and learning rate (0.003).
Observe: Compare final loss and sample quality across the three runs.
Lesson: Bigger isn't always better. The 512-unit model may not outperform 128 if there isn't enough data or training steps to justify the extra capacity.
Exercise 6: Code vs Prose
Train on a block of Python/JavaScript code (500+ chars). Then train on a novel excerpt of similar length.
Observe: Code tends to have lower loss (more predictable patterns: indentation, brackets, keywords). Prose has higher entropy (more possible continuations at each point).
Lesson: A model's performance depends on the structure of the data, not just quantity.

Key Insights to Remember

Insight 1: Training = Optimization There's no magic. Training is just: predict → measure error → adjust weights → repeat. The "intelligence" emerges from doing this billions of times on vast data.
Insight 2: Data > Architecture A simple model with lots of data almost always beats a complex model with little data. This is why GPT models are trained on terabytes of text.
Insight 3: Loss is Not Everything Low loss on training data doesn't guarantee good output. Always check the actual samples. A model can memorize (overfit) and have perfect loss but produce boring, repetitive text.
Insight 4: Hyperparameters Matter a Lot The same model architecture can perform brilliantly or terribly depending on learning rate, batch size, and other settings. ML engineering is largely about finding good hyperparameters.
Insight 5: All LLMs Are Next-Token Predictors GPT, Claude, Gemini — they all just predict the next token. The magic of "understanding" and "reasoning" emerges from this simple objective applied at enormous scale.
Insight 6: Temperature Controls Creativity vs Accuracy This is why the same AI can give you a precise factual answer (low temp) or a creative story (high temp). You're literally changing how it samples from its probability distribution.

What to Do Next

Now that you understand the basics, here's a learning path:

  1. Complete all 6 exercises above — they build intuition faster than reading alone.
  2. Read about Transformers — the architecture that replaced LSTMs. Key concept: self-attention lets the model look at all positions at once, not just left-to-right.
  3. Try a real training framework — use Google Colab (free GPU) with PyTorch to train a GPT-2 small model on your own text.
  4. Study tokenization — learn about BPE (Byte-Pair Encoding), the tokenizer used by GPT. It's a fascinating algorithm.
  5. Explore fine-tuning — take a pretrained model and adapt it to a specific task. This is how most real applications work.
  6. Learn about RLHF — Reinforcement Learning from Human Feedback is what makes ChatGPT "helpful." It's the training stage after next-token prediction.
